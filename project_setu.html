SETU Network Analytics Engine

An asynchronous text mining pipeline and graph analytics architecture featuring GPU-accelerated NLP tokenization, deterministic alias normalization, and modularity-based community detection.

Engine Repository Dataset Repository

Project Overview

SETU is a high-throughput text mining and network science platform designed to ingest unstructured natural language corpora and construct multi-dimensional, weighted graph visualizations. The engine uses a completely decoupled architecture, separating its core computation layers from relational graph data asset storage.

To validate the framework's semantic processing capacity, a comparative macro-structural study was executed across 14 extensive historical, epic, and religious texts. The system extracted thousands of relational co-occurrences, tracking how structural properties like network diameter, path lengths, density, and centralization profiles shift across different cultural text worlds.

Core Engine Engineering

  • GPU-Accelerated Core NLP: Integrated a middleweight spaCy processing pipeline with PyTorch CUDA hooks to offload batch parsing tasks to an NVIDIA RTX 4060 laptop GPU, achieving optimal processing speeds over thousands of token windows.
  • Deterministic Identity Resolution: Injected a customized EntityRuler layer to normalize character name variations, variant historical translations, and aliases into singular, static node IDs to prevent duplicate nodes.
  • Tumbling Window Co-occurrence Matrix: Engineered a sliding-window text parser that scans target streams based on word proximity, tracking overlapping occurrences into sorted tuple schemas while ignoring cross-chapter line noise.
  • Modularity-Based Partition Clustering: Deployed Louvain optimization algorithms to compute structural centrality vectors and partition the network topology into stable narrative communities.

Empirical Case Study Insights

  • Indic Cluster Topology: Revealed a distinct, highly decentralized network structure ($Density = 0.19580$) split into clear, localized text factions. The sub-graphs group into distinct clusters (Vedic, Epic, Puranic) driven by heavy cross-referencing around massive central hubs like Krishna and Arjuna.
  • Abrahamic Cluster Topology: Exhibited an integrated, linear path trace ($Density = 0.12143$) driven by chronological ancestral lines, structural continuity, and persistent references to an anchor entity across generational links.
